---
title: "Settings Reference"
description: "Complete reference for all Baritone Settings fields accessible via the Java API."
---

`BaritoneAPI.getSettings()` returns the singleton `Settings` instance. Settings are shared across all `IBaritone` instances and persist to `minecraft/baritone/settings.txt` automatically.

## Reading and writing settings

Access any setting by reading or writing its `.value` field:

```java
import baritone.api.BaritoneAPI;
import baritone.api.Settings;

Settings s = BaritoneAPI.getSettings();

// Boolean setting
s.allowSprint.value = true;

// Long setting (milliseconds)
s.primaryTimeoutMS.value = 2000L;

// Double setting
s.blockPlacementPenalty.value = 0.5;

// Integer setting
s.followRadius.value = 5;

// List setting
s.acceptableThrowawayItems.value = Arrays.asList(
    Blocks.COBBLESTONE.asItem(),
    Blocks.DIRT.asItem()
);
```

To reset a setting to its default:

```java
BaritoneAPI.getSettings().allowSprint.reset();
```

## Movement

Controls what movement types Baritone can and cannot use.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `allowBreak` | `Boolean` | `true` | Allow Baritone to break blocks while pathfinding. |
| `allowBreakAnyway` | `List<Block>` | `[]` | Blocks Baritone may break even when `allowBreak` is `false`. |
| `allowSprint` | `Boolean` | `true` | Allow sprinting. |
| `allowPlace` | `Boolean` | `true` | Allow placing blocks (bridging, scaffolding). |
| `allowPlaceInFluidsSource` | `Boolean` | `true` | Allow placing blocks inside fluid source blocks. |
| `allowPlaceInFluidsFlow` | `Boolean` | `true` | Allow placing blocks inside flowing fluid. |
| `allowParkour` | `Boolean` | `false` | Allow parkour jumps (gap jumps). Unreliable — can overshoot. |
| `allowParkourPlace` | `Boolean` | `false` | Allow placing a block mid-air to land on during parkour. |
| `allowParkourAscend` | `Boolean` | `true` | Allow sprint-jumping up to a higher block. Only active when `allowParkour` is also `true`. |
| `allowDiagonalDescend` | `Boolean` | `false` | Allow descending diagonally. Slightly unsafe in the Nether. |
| `allowDiagonalAscend` | `Boolean` | `false` | Allow ascending diagonally. Safer than diagonal descend. |
| `allowDownward` | `Boolean` | `true` | Allow mining the block directly below the player's feet. Disable to force staircases. |
| `allowVines` | `Boolean` | `false` | Enable advanced vine climbing behaviors. Generally not needed; can cause the player to get stuck. |
| `allowWalkOnBottomSlab` | `Boolean` | `true` | Allow walking on bottom-half slabs. Disable for higher reliability on paths without slabs. |
| `allowWaterBucketFall` | `Boolean` | `true` | Allow falling arbitrary distances and placing a water bucket to land safely. |
| `allowJumpAtBuildLimit` | `Boolean` | `false` | Allow parkour jumps at world build height (Y=256). Disabled by default due to server compatibility issues. |
| `assumeWalkOnWater` | `Boolean` | `false` | Treat water surface as walkable (for mods providing Jesus/water-walk functionality). |
| `assumeWalkOnLava` | `Boolean` | `false` | Treat lava surface as walkable. |
| `assumeStep` | `Boolean` | `false` | Assume step-assist; skip jumping on 1-block ascents. |
| `assumeSafeWalk` | `Boolean` | `false` | Assume safe-walk (no sneaking needed on edges). |
| `sprintAscends` | `Boolean` | `true` | Sprint-jump a block early on ascend movements where possible. |
| `sprintInWater` | `Boolean` | `true` | Continue sprinting while in water. |
| `allowWalkOnMagmaBlocks` | `Boolean` | `false` | Allow walking on magma blocks (normally Baritone sneaks to avoid damage). |
| `maxFallHeightNoWater` | `Integer` | `3` | Maximum blocks to fall onto solid ground without a water bucket. 3 = no fall damage unarmored. |
| `maxFallHeightBucket` | `Integer` | `20` | Maximum blocks to fall when a water bucket is available. |
| `allowOvershootDiagonalDescend` | `Boolean` | `true` | Allow sprinting through a descend followed by a diagonal, slightly overshooting the landing. |

```java
// Enable parkour for faster movement
Settings s = BaritoneAPI.getSettings();
s.allowParkour.value = true;
s.allowParkourPlace.value = true;
s.allowDiagonalAscend.value = true;
```

## Pathfinding

Controls the A\* pathfinder's timing, cost calculation, and behavior.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `primaryTimeoutMS` | `Long` | `500` | Stop path calculation after this many milliseconds if a valid path was found. |
| `failureTimeoutMS` | `Long` | `2000` | Hard upper limit on path calculation time even if no path is found. |
| `planAheadPrimaryTimeoutMS` | `Long` | `4000` | Primary timeout for planning-ahead calculations (while a segment is executing). |
| `planAheadFailureTimeoutMS` | `Long` | `5000` | Failure timeout for planning-ahead calculations. |
| `costHeuristic` | `Double` | `3.563` | A\* heuristic multiplier. Values below 3.5 are always admissible. Higher values compute faster but may produce suboptimal paths. |
| `pathCutoffFactor` | `Double` | `0.9` | Cut off the last `(1 - factor)` portion of every calculated path. |
| `pathCutoffMinimumLength` | `Integer` | `30` | Only apply the cutoff factor to paths of at least this many movements. |
| `planningTickLookahead` | `Integer` | `150` | Begin calculating the next path segment when the remaining tick estimate drops below this value. |
| `movementTimeoutTicks` | `Integer` | `100` | Cancel a movement if it takes more than this many ticks beyond its estimated cost. |
| `maxCostIncrease` | `Double` | `10.0` | Cancel and recalculate if a movement's cost increases by more than this amount at execution time. |
| `costVerificationLookahead` | `Integer` | `5` | Stop the path 5 movements before any step that became cost-infinite (e.g., lava spread). |
| `pathingMaxChunkBorderFetch` | `Integer` | `50` | Stop pathfinding after this many accesses to unloaded/uncached chunks. |
| `backtrackCostFavoringCoefficient` | `Double` | `0.5` | Multiply the cost of revisiting already-visited nodes. Lower = favors unexplored routes. Set to `1.0` to disable. |
| `cutoffAtLoadBoundary` | `Boolean` | `false` | Trim the path to only the loaded-chunk portion after calculation. |
| `simplifyUnloadedYCoord` | `Boolean` | `true` | Replace a `GoalBlock` in an unloaded chunk with `GoalXZ` until close enough for Y to matter. |
| `splicePath` | `Boolean` | `true` | Splice a newly calculated segment onto the current path instead of waiting for the current to finish. |
| `chunkCaching` | `Boolean` | `true` | Cache chunks in a simplified 2-bit format for long-distance pathfinding. |
| `repackOnAnyBlockChange` | `Boolean` | `true` | Repack a chunk's cache data whenever any block in it changes. |
| `slowPath` | `Boolean` | `false` | Debug mode — process one node every `slowPathTimeDelayMS` milliseconds. |
| `slowPathTimeDelayMS` | `Long` | `100` | Milliseconds between nodes in slow-path debug mode. |
| `antiCheatCompatibility` | `Boolean` | `true` | Adjust behavior to avoid tripping common anticheats (e.g., correct sprint direction). |

```java
// Give the pather more time on complex terrain
Settings s = BaritoneAPI.getSettings();
s.primaryTimeoutMS.value = 1000L;
s.failureTimeoutMS.value = 5000L;
```

## Block interaction

Controls costs and restrictions around breaking and placing blocks.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `blockPlacementPenalty` | `Double` | `20.0` | Extra cost added to paths that require placing a block. Decrease to make Baritone place blocks more willingly. |
| `blockBreakAdditionalPenalty` | `Double` | `2.0` | Extra cost for breaking any block, as a tiebreaker to prefer routes that avoid breaking. |
| `jumpPenalty` | `Double` | `2.0` | Extra cost for any movement that requires pressing jump (ascend, pillar, parkour). |
| `walkOnWaterOnePenalty` | `Double` | `3.0` | Extra cost for walking on water (uses hunger quickly). |
| `acceptableThrowawayItems` | `List<Item>` | `[dirt, cobblestone, netherrack, stone]` | Items Baritone may place as scaffolding or bridging material. |
| `blocksToAvoid` | `List<Block>` | `[tripwire]` | Blocks Baritone avoids walking on or near. |
| `blocksToDisallowBreaking` | `List<Block>` | `[]` | Blocks Baritone will never break. |
| `blocksToAvoidBreaking` | `List<Block>` | `[crafting_table, furnace, chest, trapped_chest]` | Blocks Baritone avoids breaking but will break if necessary. |
| `avoidBreakingMultiplier` | `Double` | `0.1` | Multiply break cost for `blocksToAvoidBreaking`. Values above `1.0` encourage breaking instead. |
| `avoidUpdatingFallingBlocks` | `Boolean` | `true` | Never break a block adjacent to unsupported sand or gravel to avoid cascade falls. |
| `autoTool` | `Boolean` | `true` | Automatically switch to the best tool before breaking a block. |
| `assumeExternalAutoTool` | `Boolean` | `false` | Assume another mod provides auto-tool; Baritone factors it into cost calculation but won't switch tools itself. |
| `considerPotionEffects` | `Boolean` | `true` | Adjust break costs for active potion effects (Haste, Mining Fatigue). |
| `walkWhileBreaking` | `Boolean` | `true` | Continue walking forward while breaking blocks in the path. |
| `rightClickSpeed` | `Integer` | `4` | Minimum ticks between right-clicks. |
| `blockReachDistance` | `Float` | `4.5` | Player block reach distance in blocks. |
| `itemSaver` | `Boolean` | `false` | Stop using a tool before it breaks. |
| `itemSaverThreshold` | `Integer` | `10` | Durability at which `itemSaver` stops using a tool. |
| `preferSilkTouch` | `Boolean` | `false` | Always prefer silk touch tools over regular tools of equivalent speed. |
| `strictLiquidCheck` | `Boolean` | `false` | Refuse to break blocks adjacent to any liquid. Enable if using custom fluid physics mods. |
| `pauseMiningForFallingBlocks` | `Boolean` | `true` | Wait for falling blocks to settle before continuing movement after a mine action. |
| `backfill` | `Boolean` | `false` | Place blocks behind the player to fill in tunnels as it moves. |
| `useSwordToMine` | `Boolean` | `true` | Allow using a sword to mine blocks (e.g., cobwebs). |

```java
// Make Baritone more willing to bridge over gaps
BaritoneAPI.getSettings().blockPlacementPenalty.value = 5.0;

// Never break chests or valuable blocks
BaritoneAPI.getSettings().blocksToDisallowBreaking.value =
    Arrays.asList(Blocks.CHEST, Blocks.ENDER_CHEST, Blocks.BEACON);
```

## Mining

Settings specific to `IMineProcess` behavior.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `legitMine` | `Boolean` | `false` | Only mine ores the player can actually see — no X-ray through the chunk cache. Forces exploring to find ores. |
| `legitMineYLevel` | `Integer` | `-59` | Y level to strip-mine at when `legitMine` is enabled. |
| `legitMineIncludeDiagonals` | `Boolean` | `false` | Expand legit mine vision to include diagonally-adjacent blocks. |
| `mineScanDroppedItems` | `Boolean` | `true` | Count dropped item entities of the target type as valid pathing goals (pick up ore drops). |
| `mineGoalUpdateInterval` | `Integer` | `5` | Rescan for ore goal locations every this many ticks. |
| `maxCachedWorldScanCount` | `Integer` | `10` | Stop expanding the chunk cache scan after finding this many target blocks. |
| `mineMaxOreLocationsCount` | `Integer` | `64` | Maximum number of ore target locations tracked at once. |
| `minYLevelWhileMining` | `Integer` | `0` | Refuse to mine below this Y level. Set to `0` to disable the lower bound. |
| `maxYLevelWhileMining` | `Integer` | `2031` | Refuse to mine above this Y level. |
| `allowOnlyExposedOres` | `Boolean` | `false` | Only target ores that have at least one exposed face. Useful against ore obfuscation plugins. |
| `allowOnlyExposedOresDistance` | `Integer` | `1` | Search radius around each ore when `allowOnlyExposedOres` is enabled. Keep this low. |
| `forceInternalMining` | `Boolean` | `true` | When two adjacent blocks are both targets, use `GoalBlock` or shift the goal to mine both in one pass. |
| `internalMiningAirException` | `Boolean` | `true` | Only apply `forceInternalMining` when the adjacent block is not already air. |
| `mineDropLoiterDurationMSThanksLouca` | `Long` | `250` | Milliseconds to wait after mining a block for a drop to appear before moving on. |
| `cancelOnGoalInvalidation` | `Boolean` | `true` | Cancel the current path immediately when the target block is mined and that position is no longer a goal. Speeds up mining but may miss some drops. |
| `extendCacheOnThreshold` | `Boolean` | `false` | If cache scan returns fewer results than the max but above zero, also scan the live world. CPU-intensive. |
| `blacklistClosestOnFailure` | `Boolean` | `true` | When pathfinding to a block fails, mark it unreachable and try the next closest. |

```java
// Mine legit — no X-ray, strip mine at Y=-59
Settings s = BaritoneAPI.getSettings();
s.legitMine.value = true;
s.legitMineYLevel.value = -59;
```

## Building

Settings that control `IBuilderProcess` behavior.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `buildInLayers` | `Boolean` | `false` | Build one horizontal layer at a time before moving to the next. |
| `layerOrder` | `Boolean` | `false` | `false` = bottom to top; `true` = top to bottom. |
| `layerHeight` | `Integer` | `1` | Height in blocks of each build layer when `buildInLayers` is enabled. |
| `startAtLayer` | `Integer` | `0` | Begin construction at this layer index. Useful for resuming large builds. |
| `skipFailedLayers` | `Boolean` | `false` | Skip a layer entirely if it cannot be built, and continue with the next. |
| `buildRepeat` | `Vec3i` | `(0,0,0)` | Offset to move before repeating the build. Set any axis to `0` to disable that axis. |
| `buildRepeatCount` | `Integer` | `-1` | Number of times to repeat the build. `-1` = infinite. |
| `buildRepeatSneaky` | `Boolean` | `true` | Do not notify the schematic of the movement offset on repeat — replaces the same coordinates each time. |
| `buildIgnoreExisting` | `Boolean` | `false` | Treat all existing non-air blocks as correct; only place blocks, never break. |
| `buildIgnoreDirection` | `Boolean` | `false` | Ignore facing direction of directional blocks (glazed terracotta, etc.). |
| `buildIgnoreBlocks` | `List<Block>` | `[]` | Treat these blocks as air when the schematic calls for air. |
| `buildSkipBlocks` | `List<Block>` | `[]` | Accept any block listed here as correct regardless of what the schematic asks for. |
| `buildValidSubstitutes` | `Map<Block, List<Block>>` | `{}` | Map of schematic blocks → acceptable substitutes at that position. |
| `buildSubstitutes` | `Map<Block, List<Block>>` | `{}` | Map of schematic blocks → blocks to place instead (first placeable one is used). |
| `okIfAir` | `List<Block>` | `[]` | Blocks that must become air — nothing but air is accepted at those positions. |
| `okIfWater` | `Boolean` | `false` | Do not attempt to correct blocks that are currently water. |
| `mapArtMode` | `Boolean` | `false` | Only care about the top block of each column (for map art builds). |
| `buildSchematicRotation` | `Rotation` | `NONE` | Rotate the schematic before building: `NONE`, `CLOCKWISE_90`, `CLOCKWISE_180`, `COUNTERCLOCKWISE_90`. |
| `buildSchematicMirror` | `Mirror` | `NONE` | Mirror the schematic: `NONE`, `FRONT_BACK`, `LEFT_RIGHT`. |
| `schematicOrientationX` | `Boolean` | `false` | Use the highest X coordinate as origin instead of the lowest. |
| `schematicOrientationY` | `Boolean` | `false` | Use the highest Y coordinate as origin instead of the lowest. |
| `schematicOrientationZ` | `Boolean` | `false` | Use the highest Z coordinate as origin instead of the lowest. |
| `schematicFallbackExtension` | `String` | `"schematic"` | File extension assumed when none is specified in `#build`. |
| `breakFromAbove` | `Boolean` | `false` | Experimental: allow standing above a block to mine it during building. |
| `incorrectSize` | `Integer` | `100` | Maximum number of incorrect block positions tracked at once. |
| `breakCorrectBlockPenaltyMultiplier` | `Double` | `10.0` | Multiply break cost for blocks that are already correct in the schematic. |
| `placeIncorrectBlockPenaltyMultiplier` | `Double` | `2.0` | Multiply placement cost for blocks that are incorrect in the schematic. |
| `distanceTrim` | `Boolean` | `true` | Trim incorrect positions that are far away. Improves performance on large schematics. |

```java
// Build a schematic in layers, bottom to top
Settings s = BaritoneAPI.getSettings();
s.buildInLayers.value = true;
s.layerOrder.value = false;
s.layerHeight.value = 2;

// Repeat a build 10 times, offset by 20 blocks on the X axis
s.buildRepeat.value = new Vec3i(20, 0, 0);
s.buildRepeatCount.value = 10;
```

## Following

Settings for `IFollowProcess`.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `followRadius` | `Integer` | `3` | How close (in blocks) Baritone must get to the follow target. Passed to `GoalNear`. |
| `followOffsetDistance` | `Double` | `0.0` | Maintain this distance *away* from the target, in the direction given by `followOffsetDirection`. |
| `followOffsetDirection` | `Float` | `0.0` | Direction (in degrees) of the follow offset relative to the target. |
| `followTargetMaxDistance` | `Integer` | `0` | Stop following if the target is farther than this many blocks away. `0` = no limit. |

```java
// Stay 5 blocks behind the target
Settings s = BaritoneAPI.getSettings();
s.followRadius.value = 1;
s.followOffsetDistance.value = 5.0;
s.followOffsetDirection.value = 180.0f; // behind the target
```

Settings s = BaritoneAPI.getSettings();

| `replantNetherWart` | `Boolean` | `false` | Also replant nether wart during farming (requires `replantCrops`). |
| `farmMaxScanSize` | `Integer` | `256` | Maximum number of blocks the farm process scans per cycle. |

## Elytra

Settings for `IElytraProcess` flight automation.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `elytraTermsAccepted` | `Boolean` | `false` | Must be `true` before the elytra process will activate. |
| `elytraSimulationTicks` | `Integer` | `20` | Ticks of flight to simulate per calculation. Higher = more accurate but slower. |
| `elytraPitchRange` | `Integer` | `25` | Maximum pitch deviation from a direct line-of-sight allowed during simulation. |
| `elytraFireworkSpeed` | `Double` | `1.2` | Minimum speed (blocks/tick) before auto-deploying a firework. |
| `elytraFireworkSetbackUseDelay` | `Integer` | `15` | Ticks to wait after a server position setback before deploying a firework. |
| `elytraMinimumAvoidance` | `Double` | `0.2` | Minimum padding added to the player hitbox for obstacle avoidance. |
